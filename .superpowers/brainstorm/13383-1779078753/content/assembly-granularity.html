How much assembly structure should converter extract?

This decides whether generated JS is one opaque shape or a user-editable part list.

A

Single imported shape

Generate one line that renders the combined STEP result, roughly current import behavior.

Pros

  • Most like existing reader.OneShape(); fastest to ship.
  • Lowest chance of breaking weird STEP files with incomplete names/assemblies.
  • Simple code for users who only need visual reference.

Cons

  • User cannot hide, rename, recolor, or reposition individual parts.
  • Does not use user’s knowledge of how assembly parts work together.
  • Hard to build helpful JS beyond sceneShapes.push(externalShapes[name]).
B

Extract named parts/solids into editable manifest Recommended

Generate JS with a manifest of detected parts: names, labels, colors, transforms, visibility, and comments for user edits.

Pros

  • Matches stated need: user knows assembly/parts, just not code.
  • Preserves exact STEP geometry while making organization editable in JS.
  • Good UX path: user can rename bracket, shaft, housing without reconstructing CAD.
  • Reliable if fallback keeps original combined shape when structure extraction is weak.

Cons

  • Requires deeper STEP/XDE label traversal than current OneShape() path.
  • Some STEP files lack meaningful product names; converter must generate stable fallback names.
  • More generated code and more UI decisions than one opaque shape.
C

Ask user to manually define parts after import

Render the STEP, then provide a UI or code scaffold where user groups solids and assigns names.

Pros

  • Handles messy vendor STEP files where assembly labels are missing or wrong.
  • Captures human intent directly: functional groups, not just CAD product tree.
  • Can improve generated code quality for assemblies with repeated hardware.

Cons

  • Needs significant new interaction design beyond current editor/viewport flow.
  • Manual grouping is slow for large assemblies.
  • Higher implementation risk before basic conversion proves useful.
